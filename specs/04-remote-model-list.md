# 远端模型列表（Remote model list）

**文档性质**：定义“从供应商侧发现模型”的产品行为与边界。

---

## 1. 目标

减少手工录入 Model ID 的成本，让用户可以从远端返回结果中挑选并加入当前 Provider。

---

## 2. 前置条件与失败路径

- 当前 Provider 必须具备可用连接信息（至少能定位到模型列表入口）。
- 若前置条件不满足：
  - 不发起请求
  - 在弹窗内给出明确提示（例如“供应商地址为空”）

若请求失败：

- 显示错误态与错误信息
- 支持用户点击“重试”

---

## 3. 返回结果处理

- 成功返回后，将远端条目映射为本地可加入的模型项。
- 至少需要 Model ID；无有效 Model ID 的条目应丢弃。
- 若远端缺少能力字段（如上下文长度、输出上限、模态、推理标识），可按名称启发式补默认值。
- 若返回为空列表，显示空态提示。

---

## 4. 弹窗交互规则

- 支持按 Model ID 关键词筛选：
  - 关键词以空格分隔
  - 所有关键词都命中才显示
  - 不区分大小写
- 若该 Model ID 已存在于当前 Provider：
  - 该行不可勾选
  - 显示“已添加”标记
- 支持“全选当前列表”：
  - 仅作用于“当前筛选结果中且可添加”的项
  - 需要支持部分选中态
- 底部实时显示“待添加数量”。

---

## 5. 确认添加后的行为

- 将选中项追加到当前 Provider 的模型列表末尾。
- 保存后清空全局 Selection，避免后续批量操作误选。
- 关闭弹窗并反馈成功添加数量。

去重策略：

- 弹窗层面阻止“已存在项”的再次勾选。
- 不额外承诺全局唯一（若后续要求“全局唯一 Model ID”，属于产品规则升级）。

---

## 6. 与连通性测试的分工

- 远端模型列表负责“发现候选”。
- 连通性测试负责“验证候选是否可用”。
