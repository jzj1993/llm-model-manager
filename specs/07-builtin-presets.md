# 内置 Preset（Builtin presets）

**文档性质**：定义 Preset 的产品价值、内容边界与发布约束。

---

## 1. 用户价值

- **Provider preset**：减少新建 Provider 的手工输入，优先提供展示名与分 API type 的默认连接信息。
- **Model preset**：为常见模型提供更快的录入起点（模型名与能力参数建议）。
- **URL 片段建议**：为 Base URL 和 Endpoint 提供常见候选，降低拼写错误概率。

Preset 的定位是“加速填写”，不是“替代用户判断”。

---

## 2. 内容结构（产品语义）

Preset 至少覆盖两类目录：

- Provider 目录：用于“按 Provider ID + API type”带出默认连接信息。
- Model 目录：用于“按 Model ID”带出展示名与能力参数建议。

并可附带 URL 建议词表（用于输入框下拉候选）。

---

## 3. 加载失败时的产品行为

- Preset 加载失败不阻断应用主流程。
- 失败时降级为“用户手动输入模式”：
  - 仍可新增/编辑 Provider 与 Model
  - 仅失去预填与建议候选能力

---

## 4. 与表单自动填充的关系

- Preset 只在表单的指定触发点生效（如输入失焦、点选候选、切换类型）。
- 自动填充必须遵守“尊重用户已填内容”的原则。
- 详细规则见：
  - `02-provider-model-dialogs.md`
  - `10-preset-and-form-algorithms.md`

---

## 5. 发布与维护（TPM 约束）

- Preset 变更会直接影响所有用户的默认输入体验，必须记录版本说明。
- 对热门 Provider 或模型的默认值调整，应明确迁移影响：
  - 影响新建表单默认行为
  - 不自动回写用户历史已保存配置
- 大规模更新需评估加载性能与体验稳定性。

---

## 6. 与工程实现的分工

- 本规格只约束“Preset 提供什么语义、如何影响产品行为”。
- Preset 数据如何生产、转换与打包，属于工程实现自由，不影响本规格验收。
