# 应用状态与工作流（Application state & workflows）

**文档性质**：描述跨模块的产品状态与端到端流程，不重复控件细节与实现细节。

---

## 1. 全局状态（产品语义）

应用至少维护以下用户可感知状态：

- 当前工作集（Provider 与 Model 列表）
- 当前选择集（Selection）
- 各类弹窗开关（Provider、Model、远端模型列表、导出）
- 进行中状态（如模型测试中、远端列表加载中）
- 反馈状态（toast、错误提示、空状态）

---

## 2. 启动流程

1. 读取本地工作集并规范化。
2. 尝试加载 Preset 与建议词表。
3. 进入可编辑状态。

若 Preset 加载失败，应用应降级为“手动输入模式”，不阻断主流程。

---

## 3. 持久化总规则

- 对工作集的任何结构性变更（新增、编辑、删除、排序、批量加入）都应保存到本地。
- 保存失败必须可见；失败不应导致界面静默误导。

---

## 4. 关键工作流（端到端）

### 4.1 新建/编辑 Provider

- 打开弹窗 -> 编辑字段 -> 触发 Preset 补全规则 -> 保存。
- 保存失败时保持弹窗上下文，便于用户修正。

### 4.2 新建/编辑 Model

- 打开弹窗 -> 填写或从候选选择 Model ID -> 按规则补全可选字段 -> 保存。
- 保存后该模型测试状态重置为待测。

### 4.3 删除

- 删除 Provider 或 Model 前都要求确认。
- 删除后清空 Selection，避免后续批量操作误作用到错误对象。

### 4.4 批量选择与批量测试

- Selection 支持全选、按 Provider 选、单行选。
- 批量测试前自动展开相关 Provider，便于观察结果。
- 同一模型测试进行中时忽略重复触发。

### 4.5 远端模型列表

- 进入弹窗加载态 -> 成功显示列表或失败显示错误态。
- 用户筛选、勾选、确认添加后，追加到当前 Provider 并保存。

### 4.6 导出

- 打开导出弹窗后按当前 Selection 生成内容。
- 切换导出目标时实时重算预览内容。
- 运行类动作按风险策略执行（见 `05` 与 `12`）。

### 4.7 JSON 备份与恢复

- 备份：导出当前完整工作集。
- 恢复：校验后整库替换；失败时不得改动现有数据。

---

## 5. 跨流程一致性规则

- 无可选对象时，批量动作入口禁用。
- 任一流程失败都应给出可读反馈，不吞错。
- 任何自动化行为都不得在无用户触发时执行高风险外部动作。
