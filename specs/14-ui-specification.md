# 界面规格（UI specification）

**文档性质**：只描述用户可见界面与状态，不描述组件库与实现方式。

---

## 1. 总体布局

- 页面由“顶部工具栏 + 主列表区 + 全局反馈区”组成。
- 主列表区支持纵向滚动。
- 全局轻提示（toast）用于展示复制成功、外部动作失败等短反馈。

---

## 2. 顶部工具栏

### 2.1 左侧动作

- 添加供应商
- 折叠全部
- 展开全部

### 2.2 右侧动作

- 全选（支持部分选中态）
- 已选数量展示
- 测试选中模型
- 导出选中模型
- 导出 JSON
- 导入 JSON

禁用规则：

- 当没有选中模型时，“测试选中模型 / 导出选中模型”禁用。

---

## 3. Provider 列表区

每个 Provider 卡片包含：

- 折叠/展开控制
- Provider 级选择框（可全选该 Provider 下模型）
- 名称与 ID 信息
- API type 标识
- Base URL / Endpoint 展示
- API Key 脱敏展示与复制入口
- 行内操作（加载远端模型、添加模型、编辑、删除、排序）

当卡片展开时，显示其 Model 列表。

---

## 4. Model 行

每条 Model 行至少包含：

- 选择框
- Model ID 展示
- 连通性测试入口与状态
- 编辑 / 删除
- 排序入口

测试状态需有明确视觉区分（待测、成功、失败、进行中）。

---

## 5. Provider 弹窗

- 用于新增或编辑 Provider。
- 字段至少包含：
  - Provider ID
  - Display name
  - Website
  - API type
  - Base URL
  - Endpoint
  - API Key
- 保存入口清晰；校验失败时给出可读提示。

---

## 6. Model 弹窗

- 用于新增或编辑 Model。
- 字段至少包含：
  - Model ID
  - Model display name
  - 可选能力参数（Context window、Max tokens、Reasoning、Input types）
- 支持通过候选与预设降低录入成本。

---

## 7. 远端模型列表弹窗

状态要求：

- 加载中
- 加载失败（可重试）
- 空结果
- 成功结果

成功态下支持：

- 按 Model ID 筛选
- 多选
- 全选当前可选项
- 标记“已添加”并禁止重复勾选
- 显示待添加数量与确认添加入口

---

## 8. 导出弹窗

- 左侧：导出目标切换
- 右侧：导出内容预览（多条）
- 每条内容支持复制
- 可运行类型显示运行/调起按钮

---

## 9. 空态与反馈

- 无 Provider 时显示空列表态（不强制特定插画样式）。
- 失败、成功、不可执行等都需有明确反馈文案。

---

## 10. 交互可达性要求（产品级）

- 主要动作可被清晰识别与触达（按钮文案/图标需有明确语义）。
- 弹窗打开后不应造成突兀焦点跳转或操作中断。
