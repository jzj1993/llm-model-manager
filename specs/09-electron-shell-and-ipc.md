# 运行环境能力边界（Runtime capabilities & boundaries）

目标：不描述具体工程通道名与实现细节，仅定义“应用可调用的系统能力”及其用户可见行为与风险边界。

---

## 1. 能力清单（产品视角）

- **本地存储**：读取与写入本机工作集（Provider 与 Model 配置）。
- **网络请求代理**：由应用发起 HTTP 请求，用于连通性测试与远端模型列表（避免浏览器环境限制导致功能不可用）。
- **打开外部链接**：在系统默认浏览器或目标应用中打开 URL（例如 Provider 的 Website、某些导出的 Deep Link）。
- **在终端执行脚本**：将导出内容以“可执行脚本”的方式交给系统终端运行（当前产品定位为桌面工具，允许此高权限行为）。
- **在浏览器运行脚本**：将一段 JavaScript 交给浏览器环境执行（用于特定导出目标的“外部配置注入”场景）。

---

## 2. 用户可见原则（必须）

- **可见性**：任何外部动作（打开链接、终端执行、浏览器执行）都必须让用户能看到“将要执行/打开的内容”，并且在失败时给出明确失败提示。
- **最小惊扰**：不在用户未点击“运行/调起”时自动执行任何外部动作。
- **失败不损坏**：外部动作失败不应破坏本地工作集；失败仅影响当前动作的结果。

---

## 3. 风险边界与责任划分

- **终端执行**与**浏览器脚本执行**属于高风险能力：可能读写用户环境、修改配置文件、触发外部网络访问。
- 产品要求：
  - 必须提示用户“自行确认内容无误后再继续”（具体确认策略见 `05-export-and-external-actions.md`）。
  - 文档中需明确：导出内容由系统生成，但最终执行责任在用户。

---

## 4. 平台约束（产品表述）

- 若某能力仅在特定 OS 可用，应在界面与文档中标明限制；不可用时应降级为“仅复制内容”。
